基本信息

这是一个驱动漏洞，目的是获取 root 权限。

系统防护：

SMEP + KASLR + security checks on the buffer address for read/write

Vulnerbilities:

  • 未初始化 堆内存
  • 条件竞争。同时修改 note 和 删除 note, 删除过程中没有 检查 buffer 地址。

数据流分析

struct slot_t {
struct rb_node node;
unsigned long addr;
};
使用红黑树 来管理 slot。根节点 slot_cache
顺序是 按照 addr 的大小 排列

// 内核 note 对象
struct note_t {
struct date_t date;
unsigned long epoch;
unsigned long magic;
void *buf;
struct hlist_node next;
};
使用 hash 列表管理
按照 epoch 来排列。

  • hlist_add_behind(&(note->next), node); 将 node 插入到 note-next 后
  • hlist_add_before(&(note->next), node);
  • iter = hlist_entry(node, struct note_t, next); //类似于 container_of(node, * struct slot_t, node);
  • HASH_BITS(notes) 计算notes 的 hash
  • hash_min(epoch, HASH_BITS(notes))
  • hlist_add_head(&(note->next), head);
  • static DEFINE_HASHTABLE(notes, 3); 定一个 hash 表， hash表 变量名是 notes
  • DECLARE_HASHTABLE(my_hash_table, HASH_TABLE_BITS); 声明一个 hash 表
